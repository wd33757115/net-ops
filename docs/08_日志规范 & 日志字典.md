## 1. 日志体系总览

| 层级 | 进程 | 典型位置 | 用途 |
|------|------|----------|------|
| 网关 | FastAPI :8000 | `.runtime/test/logs/fastapi.log` | 聊天、Workflow API |
| BFF | Django :8001 | `.runtime/test/logs/django.log` | 代理、JWT |
| 异步 | Celery Worker | `.runtime/test/logs/celery.log` | Skill、Workflow 步骤 |
| 可观测 | Langfuse :3001 | Web UI | Trace（非文本日志） |
| 审计 | PostgreSQL | `netops_audit_logs` | 合规审计 |
| 前端 | React | 浏览器 Console | SSE、API 错误 |

**三种记录勿混淆：**

1. **应用日志** — structlog 排障
2. **Langfuse Trace** — 链路可视化
3. **审计日志** — 用户操作合规

## 2. structlog 规范

### 2.1 初始化

- FastAPI：`src/gateway/main.py` → `configure_logging(LOG_LEVEL, LOG_FORMAT)`
- Celery：`src/core/celery_tasks/celery_app.py`
- 任务信号：`logging_signals.py` → `celery_task_started/finished/failed`

### 2.2 使用方式

```python
from src.core.logging import get_logger, bind_context, reset_context

log = get_logger(__name__)
log.info("workflow_step_completed", run_id=run_id, step_name=name)
```

- 事件名作为**第一个 positional 参数**（即 `event` 字段），**禁止** `log.info(event="...", ...)`
- 异常：`log.exception("chat_stream_failed", error=str(exc))`

### 2.3 ContextVar 字段

| 字段 | 说明 |
|------|------|
| `request_id` | HTTP 请求 UUID |
| `trace_id` | Langfuse trace |
| `run_id` | Workflow run UUID |
| `thread_id` | LangGraph 会话 |
| `user_id` | 当前用户 |
| `ticket_id` | ITSM 工单 |
| `celery_task_id` | Celery 任务 ID |

绑定：`bind_context(trace_id=..., run_id=...)`，用毕 `reset_context(token)`。

### 2.4 输出格式

| `LOG_FORMAT` | 场景 |
|--------------|------|
| `console` | 本地开发（彩色） |
| `json` | 生产 / 日志采集 |

## 3. 标准日志字段

| 字段 | 说明 |
|------|------|
| `timestamp` | ISO8601 UTC |
| `level` | info / warning / error |
| `event` | 稳定事件名 snake_case |
| `logger` | 模块路径 |
| `duration_ms` | 耗时（可选） |
| `error` | 错误摘要（失败时） |

## 5. 日志级别指南

| 级别 | 场景 |
|------|------|
| DEBUG | 路由候选、表达式求值（生产默认关闭） |
| INFO | 请求完成、步骤成功、启动摘要 |
| WARNING | Langfuse 不可用、Skill 部分加载失败 |
| ERROR | 步骤失败、500、Celery 异常 |
| CRITICAL | DB 完全不可用、进程无法启动 |

## 6. Langfuse 与日志关联

| 场景 | Langfuse | 文件日志 |
|------|----------|----------|
| 聊天 SSE | 父 trace + 节点 | fastapi.log |
| 聊天下 Workflow | 嵌套 `workflow:*` → `step:*` → `skill:*` | celery.log |
| 独立 Workflow API | 顶层 workflow trace | celery.log |
| 未配置 KEY | 自动降级 | 正常 |

**Trace URL：** `{LANGFUSE_HOST}/trace/{trace_id}`

**context 键：** `langfuse_trace_id`、`langfuse_workflow_root_span_id`
