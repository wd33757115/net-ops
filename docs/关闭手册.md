## 关闭顺序

正确的关闭顺序应该是：

```
1. FastAPI Gateway (API服务)
   ↓
2. Celery Worker (异步任务处理器)
   ↓
3. Streamlit Frontend (如果运行)
   ↓
4. Docker中间件服务 (RabbitMQ, Redis等)
```

## 关闭应用服务

### 1. 关闭FastAPI Gateway

**方法一：使用Ctrl+C**
在运行FastAPI的终端窗口按 `Ctrl+C`

**方法二：使用命令行关闭**

Windows:
```powershell
# 查找占用8000端口的进程
netstat -ano | findstr :8000

# 结束进程 (将PID替换为实际值)
taskkill /PID <PID> /F
```

**方法三：使用Python脚本查找并关闭**
```python
import psutil
import signal

for proc in psutil.process_iter(['pid', 'name', 'cmdline']):
    cmdline = ' '.join(proc.info['cmdline']) if proc.info['cmdline'] else ''
    if 'uvicorn' in cmdline and 'src.gateway.main:app' in cmdline:
        print(f"结束进程 PID: {proc.info['pid']}")
        proc.send_signal(signal.SIGTERM)
```

### 2. 关闭Celery Worker

**方法一：使用Ctrl+C**
在运行Celery Worker的终端窗口按 `Ctrl+C`

**方法二：使用命令行关闭**

Windows:
```powershell
# 查找celery进程
tasklist | findstr celery

# 结束进程
taskkill /IM celery.exe /F
```

**方法三：强制结束所有Celery进程**
```powershell
# 使用进程名结束
taskkill /IM python.exe /F /FI "WINDOWTITLE eq *celery*"

# 或使用PID
taskkill /PID <PID> /F
```

### 3. 关闭Streamlit前端

```powershell
# 查找streamlit进程
tasklist | findstr streamlit

# 结束进程
taskkill /IM streamlit.exe /F
```

## 关闭中间件服务

### 使用Docker Compose关闭

```bash
cd C:\Users\wangd\PycharmProjects\PythonProject\netops-agent\deployment

# 关闭所有中间件服务
docker-compose down

# 关闭并删除数据卷 (谨慎使用，会删除数据)
docker-compose down -v
```

### 单独关闭各服务

```bash
# 关闭RabbitMQ
docker stop netops-rabbitmq
docker rm netops-rabbitmq

# 关闭Redis
docker stop netops-redis
docker rm netops-redis

# 关闭PostgreSQL
docker stop netops-postgres
docker rm netops-postgres

# 关闭MinIO
docker stop netops-minio
docker rm netops-minio

# 关闭Qdrant
docker stop netops-qdrant
docker rm netops-qdrant

# 关闭Langfuse
docker stop netops-langfuse
docker rm netops-langfuse
```

### 停止所有Docker容器

```bash
# 停止所有运行中的容器
docker stop $(docker ps -aq)

# 删除所有容器
docker rm $(docker ps -aq)
```

## 清理进程

### 清理孤儿进程

```powershell
# 查找所有相关进程
Get-Process | Where-Object {$_.ProcessName -match "python|celery|uvicorn|streamlit"} | Format-Table Id, ProcessName, CPU

# 强制结束所有Python进程 (谨慎使用)
# taskkill /IM python.exe /F
```

### 清理僵尸进程

```bash
# 查找僵尸进程
tasklist /FI "STATUS eq zombie"

# 如果有僵尸进程，重启explorer.exe
taskkill /IM explorer.exe /F
start explorer.exe
```

### 清理端口占用

```powershell
# 查找端口占用
netstat -ano | findstr "8000 8001 5672 6379 5432 9000 15672"

# 结束占用端口的进程
taskkill /PID <PID> /F
```

## 常见问题处理

### 问题1：端口被占用

**错误信息**：
```
Error: [WinError 10013] 以一种访问权限不允许的方式做了一个访问套接字的尝试
```

**解决方法**：
1. 查找占用端口的进程
   ```powershell
   netstat -ano | findstr :8000
   ```
2. 结束该进程或使用其他端口
3. 如果是系统服务占用，使用管理员权限

### 问题2：Celery进程无法关闭

**解决方法**：
1. 强制结束进程
   ```powershell
   taskkill /IM python.exe /F /FI "WINDOWTITLE eq *celery*"
   ```
2. 清理RabbitMQ队列
   ```bash
   docker exec netops-rabbitmq rabbitmqctl purge_queue celery
   ```

### 问题3：Docker容器无法停止

**解决方法**：
1. 强制停止容器
   ```bash
   docker stop --signal=SIGKILL netops-rabbitmq
   ```
2. 如果仍然无法停止，重启Docker服务
   ```bash
   # Windows
   net stop com.docker.service
   net start com.docker.service
   ```

### 问题4：进程残留

**解决方法**：
```powershell
# 清理所有相关进程
Get-Process python,celery,uvicorn,streamlit -ErrorAction SilentlyContinue | Stop-Process -Force

# 验证清理结果
Get-Process python,celery,uvicorn,streamlit -ErrorAction SilentlyContinue
```

## 快速关闭脚本

创建 `stop_all.ps1`:

```powershell
Write-Host "开始关闭NetOps Multi-Agent服务..." -ForegroundColor Yellow

# 1. 关闭Python相关进程
Write-Host "关闭Python进程..." -ForegroundColor Green
Get-Process python -ErrorAction SilentlyContinue | Stop-Process -Force
Get-Process celery -ErrorAction SilentlyContinue | Stop-Process -Force

# 2. 关闭Docker中间件
Write-Host "关闭Docker中间件..." -ForegroundColor Green
cd C:\Users\wangd\PycharmProjects\PythonProject\netops-agent\deployment
docker-compose down

# 3. 验证端口已释放
Write-Host "验证端口状态..." -ForegroundColor Green
$ports = @(8000, 8001, 5672, 6379, 5432, 9000, 15672)
foreach ($port in $ports) {
    $result = netstat -ano | findstr ":$port"
    if ($result) {
        Write-Host "警告: 端口 $port 仍被占用" -ForegroundColor Red
    } else {
        Write-Host "端口 $port 已释放" -ForegroundColor Cyan
    }
}

Write-Host "所有服务已关闭!" -ForegroundColor Green
```

## 关机检查清单

关闭服务前，请确认：

- [ ] 已保存所有重要数据
- [ ] 已记录正在运行的任务
- [ ] Celery任务已完成或已转移到其他Worker
- [ ] Docker容器状态正常

---

## 注意事项

1. **按顺序关闭**：先关闭应用层，再关闭中间件
2. **确认任务完成**：Celery任务可能需要时间完成
3. **数据持久化**：Docker数据卷中的数据不会因关闭而丢失
4. **日志保留**：确保日志已正确保存
