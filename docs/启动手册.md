## 环境要求

### 基础环境
- **操作系统**: Windows 10/11 或 Linux/macOS
- **Python**: 3.10+ (推荐 3.11 或 3.12)
- **Docker**: 20.10+ (用于运行中间件服务)
- **Docker Compose**: 2.0+

### 硬件要求
- 内存: 最少 4GB (推荐 8GB+)
- 磁盘: 最少 10GB 可用空间

---

## 服务组件说明

本项目包含以下服务组件：

| 服务 | 端口 | 说明 |
|------|------|------|
| FastAPI Gateway | 8000/8001 | API网关，处理HTTP请求 |
| Celery Worker | - | 异步任务处理器 |
| RabbitMQ | 5672, 15672 | 消息队列(Celery Broker) |
| Redis | 6379 | Celery Result Backend |
| PostgreSQL | 5432 | 数据库(可选，用于生产环境) |
| MinIO | 9000, 9001 | 文件存储服务 |
| Qdrant | 6333, 6334 | 向量数据库(RAG服务) |
| Langfuse | 3000 | 可观测性平台(可选) |

## 启动应用服务

### 1. 启动Celery Worker

**Windows环境** (使用solo进程池):
```bash
cd C:\Users\wangd\PycharmProjects\PythonProject\netops-agent
.\venv\Scripts\python.exe -m celery -A src.core.celery_tasks.celery_app worker --loglevel=info -P solo
```


## 快速启动脚本

### Windows PowerShell

创建 `start_all.ps1`:

```powershell
# 启动Docker中间件
Write-Host "启动Docker中间件服务..." -ForegroundColor Green
cd C:\Users\wangd\PycharmProjects\PythonProject\netops-agent\deployment
docker-compose up -d rabbitmq redis

# 等待服务就绪
Start-Sleep -Seconds 5

# 启动Celery Worker (新窗口)
Start-Process -FilePath "powershell.exe" -ArgumentList "-NoExit", "-Command", "cd C:\Users\wangd\PycharmProjects\PythonProject\netops-agent; .\venv\Scripts\python.exe -m celery -A src.core.celery_tasks.celery_app worker --loglevel=info -P solo"

# 等待Worker启动
Start-Sleep -Seconds 5

# 启动FastAPI (新窗口)
Start-Process -FilePath "powershell.exe" -ArgumentList "-NoExit", "-Command", "cd C:\Users\wangd\PycharmProjects\PythonProject\netops-agent; .\venv\Scripts\python.exe -m uvicorn src.gateway.main:app --host 0.0.0.0 --port 8000 --reload"

Write-Host "所有服务已启动!" -ForegroundColor Cyan
Write-Host "FastAPI: http://localhost:8000" -ForegroundColor Yellow
Write-Host "API Docs: http://localhost:8000/docs" -ForegroundColor Yellow
Write-Host "RabbitMQ: http://localhost:15672" -ForegroundColor Yellow
```

## 注意事项

### Windows环境
1. Celery必须使用 `-P solo` 参数以单进程模式运行
2. 如果端口8000被占用，可使用端口8001: `--port 8001`

### 防火墙策略工具依赖
确保已安装 `openpyxl`:
```bash
pip install openpyxl
```

### RabbitMQ连接问题
如果遇到RabbitMQ连接问题，检查：
1. Docker容器是否正常运行: `docker ps`
2. RabbitMQ是否就绪: `docker logs netops-rabbitmq`
3. 防火墙设置是否允许5672和15672端口
