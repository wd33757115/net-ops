# 10 部署安装手册

> 版本：2026-05-24  
> 部署目录：`deployment/`

---

## 1. 部署架构

```
React (Nginx :3000)
    ↓ /api、/ws
Django BFF (:8001)
    ↓ httpx 代理
FastAPI (:8000，Docker 内 expose)
    ↓
PostgreSQL / Redis / RabbitMQ / MinIO / Celery Worker
Langfuse (:3001，可选)
```

---

## 2. 前置要求

| 项 | 要求 |
|----|------|
| Docker | 20.10+ |
| Docker Compose | 2.0+ |
| 内存 | ≥ 4 GB |
| 磁盘 | ≥ 10 GB |
| 网络 | 可拉取镜像；LLM 需访问 DeepSeek API |

## 3. 快速部署（Docker Compose 全栈）

### 3.1 准备环境变量

在项目根目录创建 `.env`，至少包含：

```env
DEEPSEEK_API_KEY=sk-xxx
SECRET_KEY=<随机强密钥>
JWT_SECRET_KEY=<随机强密钥>
POSTGRES_PASSWORD=<强密码>
```

完整变量见 [07_系统配置说明](./07_系统配置说明.md) 与根目录 [`.env.example`](../.env.example)。

### 3.2 构建并启动

**生产一键（推荐）：** 见 [`scripts/prod/README.md`](../scripts/prod/README.md)

```powershell
.\scripts\prod\start.ps1          # Windows：prod overlay + migrate + seed + 冒烟
```

```bash
./scripts/prod/start.sh           # Linux
```

**Makefile（开发/CI）：**

```bash
make -C deployment build
make -C deployment ps
make -C deployment logs
```

**直接 Compose：**

```bash
cd deployment
docker compose up -d --build
docker compose ps
docker compose logs -f
```

### 3.3 数据库迁移

```bash
make -C deployment django-migrate
make -C deployment init-db   # NetOps PG 业务表（如 Makefile 提供）
```

## 4. 分步部署

### 4.1 仅中间件

```bash
cd deployment
docker compose -f docker-compose.middleware.yml up -d
```

或：

```bash
make -C deployment up
```

启动：PostgreSQL、Redis、RabbitMQ、MinIO、Qdrant（可选）、Langfuse。

### 4.2 本地应用 + Docker 中间件

适合开发调试：

```bash
make -C deployment up        # 中间件
make -C deployment fastapi     # 终端 1
make -C deployment celery      # 终端 2
make -C deployment django      # 终端 3
make -C deployment react       # 终端 4
```

Windows 测试环境：

```powershell
.\scripts\test\install.ps1
.\scripts\test\start.ps1
.\scripts\test\verify.ps1
```

## 8. 数据持久化

Docker volumes（见 `docker-compose.yml`）：

| Volume | 内容 |
|--------|------|
| postgres-data | PostgreSQL |
| redis-data | Redis |
| minio-data | MinIO 对象 |
| rabbitmq-data | RabbitMQ |

**备份建议：** 定期备份 PostgreSQL volume 与 MinIO 数据；Chroma 索引在宿主机 `vectorstore/chroma_db/`（若挂载）。

## 9. 生产部署检查清单

- [ ] 修改所有默认密码（PG、MinIO、RabbitMQ、Django SECRET_KEY）
- [ ] `DJANGO_DEBUG=false`
- [ ] `LOG_FORMAT=json`
- [ ] `ENFORCE_BFF_ORIGIN=true`（Compose fastapi 服务默认）
- [ ] HTTPS 终止（Nginx / 负载均衡）
- [ ] CORS 限制为生产域名
- [ ] Celery Worker 独立扩容
- [ ] Langfuse 密钥与数据 retention 策略
- [ ] 防火墙：仅暴露 3000（或 443），不暴露 8000

## 10. 常用运维命令

```bash
# 状态
make -C deployment ps

# 日志
make -C deployment logs-app
docker compose -f deployment/docker-compose.yml logs fastapi -f

# 重启单服务
docker compose -f deployment/docker-compose.yml restart django

# 停止
make -C deployment down

# 停止并删 volume（慎用）
make -C deployment down-v

# 重建单服务
docker compose -f deployment/docker-compose.yml up -d --build django
```

## 11. Windows 注意事项

- Celery Worker 必须使用 `-P solo`
- 路径使用 `scripts\test\start.ps1` 而非 bash 脚本
- 日志在 `.runtime\test\logs\`
