## 3. Python 依赖安装

### 3.1 Poetry（推荐开发）

```bash

```

### 3.2 pip（与 Docker 一致）

```bash
python -m venv .venv
# Windows
.venv\Scripts\activate
# Linux/macOS
source .venv/bin/activate

pip install -r requirements.txt
```

**PYTHONPATH：** 运行/测试时需包含 `src`（pytest 已在 `pyproject.toml` 配置 `PYTHONPATH=src:tests`）。

## 7. 本地开发启动

### 7.1 方式 A：PowerShell 一键（Windows 测试环境）

```powershell
# 首次
.\scripts\test\install.ps1

# 启动（Docker 中间件 + 本地 FastAPI/Django/React/Celery）
.\scripts\test\start.ps1

# 停止
.\scripts\test\stop.ps1

# 验证
.\scripts\test\verify.ps1
```

日志目录：`.runtime/test/logs/`

### 7.2 方式 B：分进程手动启动

**1）中间件（Docker）：**

```bash
cd deployment
docker compose -f docker-compose.middleware.yml up -d
```

**2）FastAPI：**

```bash
uvicorn src.gateway.main:app --host 0.0.0.0 --port 8000 --reload
```

**3）Celery（Windows 需 solo + 多队列）：**

```powershell
# 推荐：专用脚本（启动前清理残留 Worker）
.\scripts\test\celery_worker.ps1
```

```bash
celery -A src.core.celery_tasks.celery_app worker -l info -P solo \
  -Q netops.default,netops.firewall,netops.device
```

`.env` 可选：`CELERY_WORKER_POOL=solo`、`CELERY_WORKER_QUEUES=netops.default,netops.firewall,netops.device`

**4）Django BFF：**

```bash
cd web/django_backend
daphne -b 0.0.0.0 -p 8001 django_backend.asgi:application
```

**5）React：**

```bash
cd web/react_frontend
npm run dev
```

### 7.3 方式 C：Makefile（deployment/）

```bash
make -C deployment up          # 仅中间件
make -C deployment fastapi     # 本地 FastAPI
make -C deployment django      # 本地 Django
make -C deployment react       # 本地 React
make -C deployment celery      # 本地 Celery
make -C deployment build       # 全栈 Docker 构建启动
```

## 9. 数据库初始化

### PostgreSQL 业务表

FastAPI 启动时自动 `create_all`（`src/infrastructure/db/models.py`）。

手动初始化 LangGraph checkpoint：

```python
from src.infrastructure.db.postgres import init_postgres_schema
init_postgres_schema()
```

### Django 用户表

```bash
cd web/django_backend
python manage.py migrate
python manage.py createsuperuser  # 可选
```

演示用户通常由 `install.ps1` / 种子脚本创建。

### RAG 索引

1. 文档放入 `knowledge_base/`
2. 删除 `vectorstore/chroma_db/`（如需全量重建）
3. 重启 FastAPI 或调用 `POST /api/knowledge/reindex/`

## 12. 常见问题

| 问题 | 处理 |
|------|------|
| Celery Windows 报错 | 使用 `-P solo`；勿重复开 Worker；见 `scripts/test/celery_worker.ps1` |
| 前端 502 / BFF upstream | FastAPI 未就绪；查 `fastapi.log`，确认 RAG/嵌入模型加载成功 |
| 附件下载 localhost | 生产设置 `PUBLIC_APP_URL=https://你的域名` |
| PG 连接失败 | 确认 middleware compose 已启动、`.env` 端口正确 |
| RAG 无结果 | 检查 `knowledge_base/` 与 Chroma 目录 |
| BFF 503 | Django 无法连 FastAPI，查 `FASTAPI_BASE_URL` |
| ModuleNotFoundError: src | 设置 `PYTHONPATH=src` 或从项目根目录运行 |
