## 2. 目录约定

```
src/                    # Python 业务代码（PYTHONPATH 含 src）
  gateway/              # FastAPI 入口
  agents/               # LangGraph Agent
  skill_system/         # Skill 引擎
  skills/               # 内置 Skill（SKILL.md）
  core/                 # Celery、Workflow、Skills 执行
  infrastructure/       # DB、MinIO
  observability/        # Langfuse、trace
web/
  django_backend/       # BFF
  react_frontend/       # 前端
tests/                  # 单元/集成测试
docs/                   # 项目文档（编号 01–11）
deployment/             # Docker Compose
scripts/                # CLI、测试环境脚本
```

**原则：**

- 新 Skill 放在 `src/skills/{skill-name}/SKILL.md`
- 新 Workflow 插件放在 `src/workflows/{category}/{id}/`
- 不在 `src/` 外散落业务逻辑

## 3. Python 编码规范

### 3.1 风格

- 行宽：**100**（black / ruff）
- 引号：双引号（ruff format）
- import 排序：ruff `I` 规则
- 类型注解：新代码推荐完整注解

### 3.2 日志

- **使用 structlog**，禁止新增 `print()` 调试
- 入口：`from src.core.logging import get_logger`
- 事件名：`snake_case` 稳定字符串，**不要用 kwarg `event=`**（与 structlog 冲突）
- 示例：

```python
log = get_logger(__name__)
log.info("workflow_step_completed", run_id=run_id, step_name=step.name)
```

- 上下文绑定：`bind_context(trace_id=..., run_id=...)`

详见 [08_日志规范 & 日志字典](./08_日志规范%20&%20日志字典.md)

### 3.3 异常

- 领域层：`src/common/exceptions.py`（`SkillExecutionError` 等）
- API 层：`src/core/exceptions.py`（`AppError`、`ErrorCode`）
- 新 API 优先 `raise AppError(...)` 或领域异常，由全局 handler 转信封
- 禁止在 handler 外裸 `HTTPException` 散落（遗留代码逐步迁移）

详见 [09_错误码定义文档](./09_错误码定义文档.md)

### 3.4 FastAPI

- 路由按域拆分：`skills_api.py`、`workflow_api.py` 等
- `request.state.request_id` 由中间件注入
- 长任务走 Celery，HTTP 快速返回 task_id / run_id

### 3.5 Celery

- 任务定义：`src/core/celery_tasks/tasks.py`
- Windows 开发：`-P solo`
- 任务内使用 `logging_signals` 注入的 context，**不要在 task 内 `reset_context()` 干扰 prerun**

### 3.6 Skill 开发

- 遵循 SKILL.md v2.0：[SKILL_CREATION_GUIDE.md](./SKILL_CREATION_GUIDE.md)
- CLI 创建：`python scripts/create_skill.py -n my-skill -d "描述"`
- 验证：`python scripts/validate_skill.py --all`

## 4. 前端规范

- 路径：`web/react_frontend/src/`
- API 调用：统一经 `utils/api.ts`，解析 `ApiError`（`utils/apiError.ts`）
- 环境变量：`VITE_API_BASE_URL=/api`（见 `.env.example`）
- 流式聊天：SSE 解析 `trace_start`、`final_answer`、`error` 事件
- **响应式布局**：断点 768px；统一使用 `hooks/useBreakpoint.ts`（勿在各页面重复 `matchMedia`）
- **Shell 布局**：页面级路由包裹 `GrokShellLayout`；移动端侧栏走 Drawer（`ChatShellSidebar embedded`）
- **样式约定**：移动端专用规则集中在 `index.css` 的 `@media (max-width: 767px)` 块，避免页面内联宽度魔法数

## 5. Git 与提交

- 提交信息：**简体中文**，说明「为什么」
- 不提交：`.env`、密钥、`db.sqlite3`（若含敏感数据）、`.runtime/` 日志
- 不大范围格式化无关文件

## 6. 测试规范

```bash
# 全量（需 pytest 环境）
pytest tests/ -q

# 分模块
pytest tests/skill_system/ -q
pytest tests/unit/ -q
pytest tests/observability/ -q
```

**标记：**

- `@pytest.mark.integration` — 集成测试
- `@pytest.mark.postgres` — 依赖 PG
- `@pytest.mark.celery` — 依赖 Celery

**原则：** 只写覆盖真实行为的测试；不测试 trivial getter。

## 8. 观测字段传递（Workflow / Skill）

内部字段，**不得**传入 Skill CLI：

- `langfuse_trace_id`
- `langfuse_workflow_root_span_id`
- `parent_trace_id`

使用 `strip_observability_keys()`（`src/observability/trace_context.py`）。
