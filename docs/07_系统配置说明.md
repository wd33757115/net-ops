# 07 系统配置说明

> 版本：2026-05-24  
> 配置加载：仓库根目录 `.env`（FastAPI / Celery / Django 共用）

## 1. 配置来源

| 组件 | 加载方式 | 文件 |

### 2.4 Celery

| 变量 | 默认 | 说明 |
|------|------|------|
| `CELERY_BROKER_URL` | `redis://localhost:6379/0` | 消息 broker（生产常用 RabbitMQ） |
| `CELERY_RESULT_BACKEND` | `redis://localhost:6379/1` | 结果 backend |
| `CELERY_TASK_SOFT_TIME_LIMIT` | `300` | 软超时（秒） |
| `CELERY_TASK_TIME_LIMIT` | `360` | 硬超时（秒） |
| `CELERY_MAX_RETRIES` | `3` | 默认重试次数 |
| `CELERY_WORKER_POOL` | 空（Windows 自动 `solo`） | Worker 进程池，Windows 必须 `solo` |
| `CELERY_WORKER_QUEUES` | `netops.default,netops.firewall,netops.device` | Worker 监听的队列，逗号分隔 |

**RabbitMQ 示例：**

```env
CELERY_BROKER_URL=amqp://guest:guest@localhost:5672//
CELERY_RESULT_BACKEND=redis://localhost:6379/1
CELERY_WORKER_POOL=solo
CELERY_WORKER_QUEUES=netops.default,netops.firewall,netops.device
```

本地 Windows 推荐：`.\scripts\test\celery_worker.ps1` 或 `.\scripts\test\start.ps1`（已带 `-P solo` 与 `-Q`）。

### 2.14 Redis Streams EventBus

| 变量 | 默认 | 说明 |
|------|------|------|
| `EVENT_BUS_ENABLED` | `true` | 启用 Redis Streams 事件总线 |
| `EVENT_BUS_STREAM_MAXLEN` | `100000` | Stream 最大长度 |
| `EVENT_BUS_CONSUMER_BATCH_SIZE` | `20` | 消费者批量读取 |
| `EVENT_BUS_CONSUMER_BLOCK_MS` | `500` | XREAD BLOCK 毫秒 |
| `EVENT_BUS_POLL_IDLE_SEC` | `1.0` | 轮询空闲间隔 |
| `EVENT_BUS_DIRECT_NOTIFY_FALLBACK` | `true` | Consumer 未启动时仍写 DB 站内通知 |

## 5. Docker Compose 覆盖

`deployment/docker-compose.yml` 中为各服务注入环境变量，典型覆盖：

- FastAPI：`ENFORCE_BFF_ORIGIN=true`
- PostgreSQL：容器内 hostname `postgres`
- Langfuse：独立 PostgreSQL + 初始化

修改 Compose 后需 `docker compose up -d` 重建。

---

## 6. 配置优先级

1. 进程环境变量（最高）
2. 仓库根 `.env`
3. `web/django_backend/.env`（Django 本地覆盖）
4. Pydantic / Django 代码默认值
