# 公文写作技能

本技能提供党政机关公文写作的完整指南，涵盖格式规范、常用模板、写作技巧和质量检查，确保公文格式正确、语言规范、结构清晰。

## 技能概览

当用户使用本技能时，将会：
- 查阅GB/T 9704-2012《党政机关公文格式》国家标准
- 根据公文文种选择合适的模板
- 参考语言规范和写作技巧
- 使用质量检查清单核实公文质量

## 技能使用场景

本技能适用于处理以下公文写作需求：

### 1. 撰写新公文
当用户需要新写一份公文时，协助选择文种、提供模板、指导写作。

**操作流程**：
1. 了解公文用途和背景
2. 确定公文文种（请示、通知、函、总结、纪要等）
3. 提供相应模板
4. 指导填写关键内容
5. 检查格式和语言规范

### 2. 修改完善公文
当用户已有公文初稿，需要修改完善时，提供修改建议。

**操作流程**：
1. 通读原文，了解基本内容
2. 对照检查清单检查问题
3. 指出格式、语言、逻辑问题
4. 提供修改建议

### 3. 检查公文格式规范
当用户已完成公文，需要检查是否符合GB/T 9704-2012标准。

**操作流程**：
1. 使用质量检查清单逐项核对
2. 检查字体、字号、层次序数等格式要素
3. 标注不符合规范的地方
4. 提供修改建议

### 4. 提供公文写作指导
当用户需要学习公文写作方法、技巧时，提供指导。

**操作流程**：
1. 了解用户具体需求
2. 提供相关参考资料
3. 讲解写作要点和注意事项
4. 提供实例分析

### 5. 审核公文质量
对公文进行全面审核，包括格式、内容、语言、逻辑等方面。

**操作流程**：
1. 使用质量检查清单全面检查
2. 分析内容完整性、逻辑严密性
3. 评估语言准确性和规范性

## 核心工作流程

### 公文撰写五步流程

```
需求确认 → 文种选择 → 模板应用 → 内容写作 → 质量检查

详细步骤：

1. 需求确认：了解公文目的、使用场景、具体内容要求
   - 询问用户：公文用途是什么？
   - 询问用户：收文对象是谁？
   - 询问用户：核心内容是什么？

2. 文种选择：根据用途选择合适的公文文种
   - 向上级请求批准：请示
   - 向下级布置工作：通知
   - 平行单位商洽工作：函
   - 总结工作情况：总结
   - 记录会议内容：纪要
   - 汇报工作情况：报告

3. 模板应用：根据文种选择对应模板
   - 参考：references/document-templates.md
   - 提供模板框架
   - 说明各部分填写要求

4. 内容写作：指导用户填写具体内容
   - 提供写作要点
   - 注意语言规范
   - 避免常见错误

5. 质量检查：对照清单检查公文质量
   - 使用：checklists/quality-checklist.md
   - 逐项核对
   - 标注需要修改的地方
```

### 快速写作指南

针对不同文种的快速写作要点：

**请示写作要点**：
- 明确请示依据（政策、实际情况）
- 说清请示事项（具体、明确）
- 一文一事，不多头主送
- 结尾："妥否，请批示"

**通知写作要点**：
- 说明通知依据（目的、意义）
- 明确通知事项（清晰、具体）
- 提出执行要求（可操作）
- 结尾："特此通知"

**函写作要点**：
- 说明发函依据（缘由）
- 明确商洽/答复事项
- 注意语气得体（平行文）
- 结尾："请予函复"或"特此函复"

**总结写作要点**：
- 概括主要成绩（数据支撑）
- 提炼经验做法（有特色）
- 查找存在问题（不回避）
- 明确今后打算（可操作）

**纪要写作要点**：
- 记录会议信息完整
- 客观记录讨论情况
- 明确议定事项
- 落实责任分工

## 格式规范指南

本技能提供的格式规范严格遵循GB/T 9704-2012《党政机关公文格式》国家标准。

### 核心格式要点

**纸张与版面**：
- 用纸规格：A4型纸（210mm×297mm）
- 天头（上白边）：37mm±1mm
- 订口（左白边）：28mm±1mm
- 版心：156mm×225mm
- 行距：28磅
- 每面：22行×28个字

**字体字号**：
| 要素 | 字体 | 字号 | 说明 |
|------|------|------|------|
| 发文机关标志 | 小标宋体 | 自定 | 红色 |
| 文件标题 | 小标宋体 | 2号 | 居中 |
| 正文 | 仿宋_GB2312 | 3号 | 主体 |
| 第一层次标题 | 黑体 | 3号 | "一、" |
| 第二层次标题 | 楷体_GB2312 | 3号 | "（一）" |
| 发文字号 | 仿宋_GB2312 | 3号 | 六角括号 |

**结构层次序数**：
```
一、第一层次（黑体）
  （一）第二层次（楷体_GB2312）
    1. 第三层次（仿宋_GB2312）
      （1）第四层次（仿宋_GB2312）
```

**日期与文号**：
- 成文日期：阿拉伯数字全称（2026年1月26日）
- 发文字号：机关代字〔年份〕序号号（如：国办发〔2026〕5号）

### 详细规范

**完整格式规范请参考**：
📄 [GB/T 9704-2012标准详解（完整版）](./references/gb-t-9704-2012-standard.md)

**内容包括**：
- 公文用纸技术要求

- 各部分格式编排规则（版头、主体、版记）
- 页码编排规范
- 特定格式要求（信函、命令、纪要）
- 公文排版检查要点

## 常用公文模板

本技能提供以下常用公文的详细模板和示例：

### 1. 请示

**用途**：向上级机关请求指示、批准

**模板要点**：
```
[发文机关]关于[事由]的请示

[主送机关]：

根据[依据]，现将有关问题请示如下：

一、[情况说明]

二、[请求事项]

妥否，请批示。

[发文机关]
[成文日期]
```

**案例示例**：
- 关于解决项目资金的请示
- 关于增加人员编制的请示
- 关于举办活动的请示

### 2. 通知

**用途**：批转、转发、传达事项，要求执行

**模板要点**：
```
[发文机关]
关于[事由]的通知

[主送机关]：

根据[依据]。现将有关事项通知如下：

一、[通知事项1]

二、[通知事项2]

特此通知。

[发文机关]
[成文日期]
```

**案例示例**：
- 会议通知
- 培训通知
- 批转转发通知

### 3. 函

**用途**：不相隶属机关之间商洽、询问、答复

**类型**：
- 发函：商洽函、询问函、请批函
- 复函：答复函、批准函

**模板要点**：
```
[发文机关]关于[事由]的函

[主送机关]：

[发函依据]。现将有关事项说明如下：

一、[事项1]

请予函复。

[发文机关]
[成文日期]
```

### 4. 总结

**用途**：对一定时期内工作的回顾和评价

**类型**：
- 工作总结
- 经验总结
- 综合总结

**模板要点**：
```
[单位名称][年份]工作总结

[引言]：
[年份]年在[上级]领导下，[总体成绩]。现将工作总结如下：

一、[成绩1]

二、[成绩2]

[存在的问题]

[今后打算]

[发文机关]
[成文日期]
```

### 5. 会议纪要

**用途**：记载会议情况和议定事项

**模板要点**：
```
[会议名称]纪要

时间：[日期]
地点：[地点]
主持人：[姓名]
出席：[人员名单]
请假：[人员名单]
列席：[人员名单]

[会议内容]

[印发机关]
[成文日期]
```

### 6. 报告

**用途**：向上级汇报工作、反映情况

**类型**：
- 工作报告
- 情况报告
- 答复报告

## 语言写作规范

公文语言应当准确、平实、简明、庄重。

### 四大原则详解

**1. 准确**
- 措辞恰当，概念准确
- 判断合乎逻辑
- 避免歧义

**2. 平实**
- 语言朴实无华
- 不堆砌辞藻
- 避免溢美虚饰

**3. 简明**
- 开门见山，直截了当
- 删繁就简，避免冗长
- 多用短句，少用长句

**4. 庄重**
- 使用规范书面语
- 不用口语、方言、网络语
- 语气态度得体

### 写作技巧

**标题拟定六看**：
- 是否准确恰当
- 是否明确集中
- 是否照应总题
- 是否合乎逻辑
- 是否具有艺术性
- 是否匀称和谐

**经验提炼四步法**：
1. 罗列工作、整理分类
2. 归纳亮点、提炼主题
3. 划分层次、打磨标题
4. 严谨行文、反复修改

**常用规范用语**：
- 开头用语：根据、为、关于
- 结尾用语：特此通知、妥否请批示、特此报告
- 期请用语：恳请、拟请、望予
