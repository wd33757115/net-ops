## 请示模板

### 使用说明

**定义**：请示是向上级机关请求指示、批准的公文文种。

**使用场景**：
1. 对上级机关的法律、法规、政策等有疑问需要明确说明
2. 工作中出现新情况、新问题，缺乏明确的政策依据
3. 超出自身权限范围事项，需要上级批准
4. 重要事项需要上级机关决策

**基本原则**：
- **一文一事**：一件请示只写一个问题
- **主送单一**：只能有一个主送上级机关
- **逐级请示**：一般不得越过直属上级机关
- **及时性**：需要及时行文，不得事后补报

---

### 标准模板

```
[发文机关] 关于 [请示事由] 的请示

[主送机关]：

[导语/依据]：
根据[法律、法规、政策依据]，[情况说明]。现将有关问题请示如下：

[主体/请示内容]：
一、[具体事项1]
[详细说明：背景、现状、问题]

二、[具体事项2]
[详细说明：请示的具体要求]

三、[必要附件说明]
[相关文件、数据、材料等附件说明]

[结尾/请求语]：
妥否，请批示。

[附件说明，如有]
附件：1.[附件名称]
      2.[附件名称]

[发文机关署名]
[成文日期]

（联系人：[姓名]，联系电话：[电话号码]）
```

### 示例一：关于请求解决资金的请示

```
青山区东风街道办事处
关于兴建便民早餐店资金缺口的请示

2026年1月26日

（联系人：张三，联系电话：12345678901）
```

**范例分析**：
- ✅ 标题：发文机关+事由+文种
- ✅ 主送机关：单一上级机关
- ✅ 行文依据：有政策文件依据
- ✅ 事项明确：说明资金缺口
- ✅ 一文一事：仅请求资金
- ✅ 请求语规范："妥否，请审核批示"

### 错误案例分析

**错误示例**：
```
东风街道办事处关于申请增设便民早餐店的请示报告

青山区政府、财政局：

根据《青山区人民政府关于推动社区"早餐工程"建设的决定》，我街道打算新增设的便民早餐店在2023年12月1日开业，产权归下属社区所有，聘请社区内的下岗职工承包经营。便民早餐店预算建设资金共计500万元，现已筹集资金300万元，还有两百万元资金没有着落，为此，要求区政府、财政局给予支持解决。

此事关系到社区居民的切身利益，务必批准。

东风街道办事处
XXXX年XX月X日
```

**错误点**：
1. ❌ 标题："请示报告"混用，应为"请示"
2. ❌ 主送机关：多个主送机关，应单一主送
3. ❌ 日期："两〇〇六年"应为"2006年"
4. ❌ 用语："打算"、"没有着落"、"要求"不规范，改为"决定"、"缺口"、"拟请"
5. ❌ 结语："务必批准"语气生硬，改为"妥否，请审批"
6. ❌ 缺少联系人信息

### 请示写作注意事项

1. **必须一文一事**
   - 不得将多个请示事项写在一起
   - 需分别行文

2. **主送机关要单一**
   - 防止多头主送、多头请示
   - 只能有一个主送机关

3. **不得越级请示**
   - 特殊情况越级请示，需同时抄送被越过的上级机关

4. **语气要得体**
   - 使用请求性、征询性语言
   - 如"妥否，请批示"、"当否，请批复"等

5. **附件要齐全**
   - 必要的政策依据、数据材料、方案等应作为附件

6. **紧急请示要标注时限**
   - 需要上级在一定期限内批复的，应注明
   - 如"以上请示，请于一月内批复"

7. **注明联系人**
   - 便于上级机关联系了解情况

## 通知模板

### 使用说明

**定义**：通知是用书面语言告知有关单位和人员某些特定事项，要求办理或知晓的公文。

**使用场景**：
1. 发布、传达要求下级机关执行的事项
2. 批转、转发公文
3. 任免人员
4. 传达性通知（会议通知、培训通知等）

**主要特点**：
- 应用范围广泛，使用频率高
- 内容具有执行性、要求性
- 时效性强

---

### 标准模板

```
[发文机关]
关于[通知事由]的通知

[主送机关]：

[通知根据]：
[制发通知的政策依据、目的、意义等]。

[通知内容]：
一、[工作安排1]
[具体措施、原则、要求]

二、[工作安排2]
[具体措施、具体内容]

……

[执行要求]：
[贯彻落实的具体安排、希望、联系方式等]

特此通知。

[发文机关署名]
[成文日期]
```

### 会议通知结构分析

**完整结构**：开会原因 + 会议内容 + 会议安排 + 会议要求

```
会议通知结构：

一、开会原因
[目的、依据、意义]

二、会议安排
- 会议名称：明确具体会议名称
- 会议时间：日期、具体时间
- 会议地点：详细地址、会议室名称
- 参会人员：明确参会范围、具体人员
- 会议议程：具体议题和安排

三、会议要求
- 报名要求：报名截止时间、方式
- 会前准备：需要携带的材料
- 会风会纪：请假制度、会议纪律
- 联系人及联系方式
```

### 标题排列形式

公文标题两行或多行排列应采用**梯形**或**菱形**：

**梯形排列**：

```
国务院批转《发展改革委关于2024年
深化经济体制改革工作意见》的通知
```

**菱形排列**：

```
国务院关于提请审议《中年人民共和国
外商投资企业和外国企业所得税法
（草案）的议案》
```

**原则**：
- 做到词意完整
- 排列对称
- 长短适宜
- 间距恰当

## 函模板

### 使用说明

**定义**：函是不相隶属机关之间商洽工作、询问和答复问题、请求批准和答复审批事项的公文。

**特点**：
- 使用范围广泛
- 形式灵活多样
- 属于平行文
- 分为发函和复函

**主要类型**：
1. 商洽函：机关间商洽工作
2. 询问函：询问有关问题
3. 答复函：答复询问事项
4. 请批函：请求批准事项
5. 批准函：批准请求事项

---

### 标准模板（发函）

```
[发文机关] 关于 [事由] 的函

[主送机关]：

[导语]：
[发函依据、目的、背景]。现将有关事项说明如下：

[正文]：
一、[事项1]
[具体说明]

二、[事项2]
[具体说明]

……

[结语]：
请予考虑/望予批准/请予函复/盼复/敬请函复/特此函告

[附件，如有]

[发文机关署名]
[成文日期]
```

---

### 标准模板（复函）

```
[发文机关]
关于 [事由] 的复函
```

### 经验提炼四步法

**第一步：罗列工作、整理分类**
- 全面罗列所有工作内容
- 按性质、重要性、时间等维度分类
- 建立工作台账，避免遗漏

**第二步：归纳亮点、提炼主题**
- 从分类工作中找出亮点和特色
- 确定总结的核心主题
- 提炼主要观点和创新点

**第三步：划分层次、打磨标题**
- 根据主题划分逻辑层次
- 为每个层次拟定恰当标题
- 主标题和小标题相互呼应

**第四步：严谨行文、反复修改**
- 文字叙述要精练，删繁就简
- 标题要盖住内容，概括准确
- 逻辑层次要清楚，论证充分
- 数据要准确，事例要典型

## 报告模板

### 使用说明

**定义**：报告是向上级机关汇报工作、反映情况、答复上级机关询问的公文。

**主要类型**：
1. 工作报告：向上级汇报工作进展、成效
2. 情况报告：向上级反映重要情况、重大问题
3. 答复报告：答复上级机关的询问
4. 报送报告：向上级报送文件或材料

**与请示的区别**：
- 报告：陈述性公文，不需要上级答复
- 请示：请求性公文，需要上级批复

```
二、存在的主要问题
（一）企业主体责任落实不到位。部分企业主要负责人安全意识淡薄，安全投入不足，安全管理制度不健全，员工培训教育走过场。

（二）监管执法效能有待提升。监管执法力量不足，专业技术人员占比较低，执法装备和技术支撑手段滞后，信息化监管水平不高。

（三）应急处置能力需要加强。应急救援队伍专业水平不高，大型、专业救援装备不足，跨部门协同联动机制不够健全。

（四）公共安全风险不容忽视。随着城市化进程加快，人员密集场所、高层建筑、老旧小区等方面的安全风险凸显。

三、下一步工作打算
（一）持续压实安全生产责任。进一步健全责任体系，严格落实"党政同责、一岗双责、齐抓共管、失职追责"和"三个必须"要求，强化督查考核和追责问责。
```
