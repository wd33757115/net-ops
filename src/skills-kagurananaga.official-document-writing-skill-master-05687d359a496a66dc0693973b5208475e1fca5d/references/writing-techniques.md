## 公文语言四大原则

### 一、准确

准确是公文语言的第一要求，包括用词准确、造句准确、表意准确。

#### 1. 措辞恰当
- 严格遵循语言规范，概念表达清晰明确
- 判断要合乎逻辑，充分辨别词语的轻重缓急
- 对有歧义的词句尤其慎用

**错误案例**：
```
据查，最近我市市民养狗成风，影响环境卫生，且多次发生疯狗咬人事件。现要求养有狗者必须于本月15日之前，将所养的狗自行处理，违反此规定者，由公安局派人捕杀。
```

**问题分析**：
- "成风"用词不当，含有贬义
- "疯狗"概念不清，应为"流浪狗或未免疫犬只"
- "自行处理"表述模糊，不明确
- "捕杀"过于强硬，使用"处置"更合适

**正确表述**：
```
近期，我市部分街区出现犬只扰民、影响环境卫生等问题。根据《XX市养犬管理条例》规定，请养犬居民于X月X日前办理犬只登记、免疫手续，并加强管理。对流浪犬只，由公安部门会同城管部门依法收容处理。
```

#### 2. 句子成分要完整
汉语的单句有主、谓、宾、定、状、补六种句子成分。公文写作中，为保证表意准确、严谨，句子成分要适当、合理，不能任意省略和无故残缺。

**错误案例**：
```
通过学习，使全体干部提高了认识。（缺少主语）
```

**正确表述**：
```
通过学习，全体干部提高了认识。
```

#### 常见用词错误的修正

| 错误用词 | 问题分析 | 正确用法 |
|---------|---------|---------|
| 截止2014年10月30日 | "截止"后面不能直接跟时间 | 截至2014年10月30日，或2014年10月30日截止 |
| 上世纪八十年代以来 | "世纪"和"年代"用汉字数字 | 20世纪80年代以来 |
| 二〇〇六年 | 公文要求用阿拉伯数字 | 2006年 |
| 老百姓 | 口语化 | 市民、群众、居民 |
| 一把手 | 口语化 | 主要负责人、主管领导 |
| 打算 | 口语化 | 决定、拟定、计划 |
| 没有着落 | 口语化 | 缺口、存在困难 |
| 要求 | 语气强硬，对上级不适用 | 拟请、恳请、建议 |
| 一把手抓 | 口语化 | 主要领导亲自抓 |
| 一通到底 | 口语化 | 一抓到底、落实到位 |
| 狠抓 | 口语感强 | 强化、加强、加大力度 |

### 二、平实

语言要朴实无华、语意实在、通俗易懂，不堆砌辞藻，不溢美，不虚饰。

**错误案例**：
```
在市委、市政府的英明领导下，全区上下万众一心、众志成城，以雷霆万钧之势、排山倒海之力，全面落实各项工作部署，取得了举世瞩目的辉煌成就。
```

**问题分析**：
- 过度堆砌形容词，言过其实
- "英明领导""举世瞩目"等词有失庄重
- 空话套话较多，实质性内容少

**正确表述**：
```
在市委、市政府的正确领导下，全区上下团结协作，认真落实各项工作部署，圆满完成了年度目标任务。
```

#### 平实语言的要求
1. **用词要准确实在**
   - 少用形容词、副词
   - 多用名词、动词，表述具体行动和事实

2. **避免华丽辞藻**
   - 不使用"极其""非常""特别"等程度副词
   - 不使用"辉煌""卓越""巨大"等夸张词语

3. **数据说话**
   - 用具体数据说明工作成效
   - 例："完成项目投资120亿元"而不是"完成项目建设取得巨大成果"

### 三、简明

公文语言要简洁明快、开门见山、言之有物，尽可能使用短句。

**错误案例**：
```
为了使本次会议精神得到全面贯彻落实，切实推动各项工作有效开展，经研究决定，特制定本实施方案。
```

**问题分析**：
- 句子过长，修饰成分过多
- "为了""使""得到""切实"等词语重复累赘

**正确表述**：
```
为贯彻本次会议精神，推动工作落实，特制定本方案。
```

#### 实现简明的具体方法

1. **开门见山**
   - 开头直接点明主题，不绕弯子
   - 例："根据上级要求，现就有关事项通知如下："

2. **删繁就简**
   - 删除不必要的词语
   - 合并重复表述
   - 删除"的""了""进行"等虚化词语

3. **使用短句**
   - 每个短句只表达一个完整意思
   - 例："会议总结了工作。分析了形势。部署了任务。"

4. **词语精准**
   - 用一个词表意清楚，不用一串词
   - 例：用"已"不用"已经"，用"拟"不用"打算"，用"即日"不用"从即日起"

---

### 四、庄重

公文语言必须郑重、严肃，语气坚定、格调凝重，使用规范的书面语言。

#### 庄重的基本要求
1. **使用规范的书面语**
   - 不用方言、土语
   - 不用网络语言、流行语
   - 不用口语、俗语

**错误案例**：
```
这事儿得好好琢磨琢磨，不能拍脑袋决策。
```

**正确表述**：
```
此事需认真研究，慎重决策。
```

2. **使用稳妥的词语**
   - 避免绝对化表态
   - 留有余地

**例如**：
- 用"应予肯定"不用"就是好"
- 用"基本符合"不用"完全符合"（未完全核实前）
- 用"原则同意"不用"完全同意"（需补充完善时）
- 用"拟同意"不用"同意"（未经最终决定时）

3. **使用专用语**

#### 公文常用专用语

**开头用语**：
- 根据、按照、遵照、依照
- 为、为了
- 关于、由于、鉴于
- 兹、兹定于、兹定于
- 欣闻、惊悉

**承启用语**：
- 现将、特作如下
- 特提出以下
- 综上所述、总而言之

**结尾用语**：
- 特此通知、特此函告
- 当否，请指示
- 妥否，请批复
- 请予审批、请予支持
- 以上报告，请审阅
- 此复、特此复函

**经办用语**：
- 经、业经、已将
- 试行、暂行
- 办理、施行、贯彻执行

**期请用语**：
- 恳请、拟请、敬请
- 盼予、希予、望予
- 务希、尚望

**判断用语**：
- 系、确系、显系、果系
- 应、本应、理当

**谦敬用语**：
- 承蒙、蒙、不胜
- 鼎力支持、大力协助

**时态用语**：
- 兹、现已、业已
- 届时、值此
- 如期、按期

## 标题拟定技巧

### 拟定标题的"六看"

#### 1. 看是否准确恰当
标题必须对本部分所写内容的主旨进行准确、恰当的概括。

**要求**：
- 题文相符，宽窄适度
- 杜绝题不对文、文不对题
- 避免过小或过大

**错误案例**：
```
标题：加强干部队伍建设
内容：只写了干部培训工作
问题：标题过大、过宽
```

**正确做法**：
```
标题：加强干部培训工作
内容：围绕培训对象、内容、方式、效果展开
```

#### 2. 看是否明确集中
拟定标题不能过大过杂，越单一、越集中越好。

**错误案例**：
```
标题：关于党建工作、业务工作、队伍建设的思考
问题：标题包含三个内容，过于杂乱
```

**正确做法**：
分开撰写或重新提炼主题：
```
标题：以高质量党建引领高质量发展
```

#### 3. 看是否照应总题
各层次的小标题与全文总标题是相互领属关系，要注意呼应配合。

**示例**：
```
总标题：推进政务服务改革 优化营商环境

一级标题：
一、优化审批流程，实现"一窗受理"
二、推进数字转型，实现"一网通办"
三、强化服务保障，实现"一次办好"

特点：都围绕"优化""服务"主题，照应总题
```

#### 4. 看是否合乎逻辑
从逻辑上讲，标题之间的关系是：
- 总标题与小标题：领属关系
- 小标题相互间：并列关系

**错误案例**：
```
一级标题：
一、加强组织领导
二、加强工作保障
三、提升业务能力

逻辑错误："组织保障"与"业务能力"不在同一层次
```

**正确做法**：
```
一级标题：
一、加强组织领导，健全工作机制
二、加大投入保障，改善基础条件
三、强化队伍建设，提升专业能力
```

#### 5. 看是否具有艺术性
标题在准确、明确的前提下，努力做到生动、有新意。

**提升技巧**：
1. **运用形象化语句**
   - 例："打通最后一公里"、"下好先手棋"

2. **运用对仗式标题**
   ```
   聚焦主责主业 强化担当作为
   坚持问题导向 狠抓整改落实
     ```

3. **运用排比式标题**
   ```
   学习要深入
   调研要扎实
   整改要彻底
   ```

4. **运用具有概括性的"原话"**
   - 选用本部分中概括性强、有代表性的话语
   - 例：使用"让数据多跑路、让群众少跑腿"概括数字化改革

#### 6. 看是否匀称和谐
标题的句式、字数要整齐协调，形成美感。

**错误案例**：
```
一、加强组织领导
二、健全工作机制，形成工作合力
三、深化改革
```

**问题**：句式不统一，长短不一

**正确做法**：
```
一、加强组织领导
二、健全工作机制
三、深化改革创新
```

### 标题拟定的"三个注重"

#### 1. 注重积累

**积累方法**：
- 从报刊杂志、公文材料中收集好标题
- 建立标题素材库，分类整理
- 重点学习上级领导讲话、政策文件

**积累内容**：
- 总标题（工作报告、总结等）
- 一级标题（各部分大标题）
- 特色小标题（亮点工作小标题）

**积累原则**：
- 记录精妙、新颖、有亮点的标题
- 记录结构清晰、逻辑严密的标题体系
- 记录运用修辞手法、富有文采的标题

#### 2. 注重模仿

**模仿方法**：
- 有合适的可以直接使用
- 不完全匹配的可以改头换面后使用
- 不能启发自己的标题不用

**模仿技巧**：
```
原文标题：深化"放管服"改革 优化营商环境

模仿示例：
1. 深化行政审批改革 提升政务服务效能
2. 推进数字化转型 提高治理现代化水平
3. 强化科技创新 赋能高质量发展

结构：动作+主题+效果
```

#### 3. 注重实践

**实践要点**：
- 先拟定标题再写内容（主题先行）
- 边写边修改标题
- 写完后再次打磨标题

**实践原则**：
- 标题不一定一次定型，可多次修改
- 从实践中提炼特色鲜明的标题
- 通过不断实践，提升标题提炼能力

## 经验提炼四步法

### 第一步：罗列工作、整理分类

**目的**：全面掌握工作内容，避免遗漏

**具体做法**：
1. 列出所有工作内容（不分大小）
2. 根据工作性质进行分类整理

**常用分类维度**：
- **按业务领域分**：业务工作、党建工作、队伍建设
- **按工作性质分**：常规工作、重点工作、创新工作
- **按工作成效分**：成绩突出、一般、有待改进
- **按时间顺序分**：第一季度、第二季度、全年工作

**示例**：
```
政务服务工作分类：
一、审批制度改革（重点）
二、服务环境优化（常规）
三、数字化建设（创新）
四、队伍建设（基础）
```

### 第二步：归纳亮点、提炼主题

**目的**：找准特色、突出重点

**具体做法**：
1. 从分类中找出有创新性、有影响力、有成效的工作
2. 确定总结的核心主题和主要观点
3. 提炼关键词和特色表述

**提炼标准**：
- **创新性**：工作有创新做法
- **示范性**：经验可复制推广
- **实效性**：取得明显成效
- **典型性**：具有代表性

**示例**：
```
亮点归纳：
常规做法：推行"一窗受理"
提炼亮点：全省首创"无差别全科受理"模式

常规做法：优化审批流程
提炼亮点：在全省率先实现企业开办"1日办结"

常规做法：加强队伍建设
提炼亮点：开展"服务明星"竞赛，满意度全国领先
```

### 第三步：划分层次、打磨标题

**目的**：建立逻辑框架，形成体系

**具体做法**：
1. 根据主题划分逻辑层次（一般3-5个层次）
2. 为每个层次拟定标题
3. 检查各层次间逻辑关系

**层次划分方法**：
```
常见结构：
一、坚持问题导向，找准改革突破口  （为什么改）
二、坚持系统思维，统筹推进改革    （怎么改）
三、坚持科技赋能，提升智慧化水平  （用什么手段）
四、坚持服务为本，提升群众满意度  （达到什么效果）

逻辑：四个"坚持"层层递进，从认识到行动到结果
```

### 第四步：严谨行文、反复修改

**目的**：确保质量，精益求精

**具体要求**：

#### 1. 文字要精练
- 删除可有可无的词语
- 删去重复意思的表述

**修改示例**：
```
原句：在参加培训的人员中，共有150名同志参加了本次培训活动。
修改：参加培训人员150名。

原句：为了进一步加强安全生产监管工作的需要
修改：为加强安全生产监管
```

#### 2. 标题要盖住内容
- 标题有足够的概括性
- 内容不超出标题范围
- 避免出现"帽子大、内容小"或"帽子小、内容大"

**错误示例**：
```
标题：加强组织协调
内容：写了组织、协调、保障、督查等多方面
问题：标题概括不了内容
```

**正确做法**：
```
标题：加强组织领导
内容：只写组织方面的内容

或改为：
标题：强化组织保障
内容：保留组织、协调、保障等方面的内容
```

#### 3. 逻辑层次要清楚
- 各层次之间不交叉、不重复
- 层次划分符合MECE原则（相互独立、完全穷尽）
- 篇章、段落、句子之间衔接自然

**逻辑检查清单**：
- [ ] 是否有内容交叉重复
- [ ] 是否有重要内容遗漏
- [ ] 层次划分是否合理
- [ ] 先后顺序是否恰当
- [ ] 过渡是否自然流畅

## 常见错误及修正

### 一、格式类错误

| 错误类型 | 错误表现 | 正确做法 |
|---------|---------|---------|
| 用纸规格 | 使用B5、16开等非标准用纸 | 必须使用A4型纸 |
| 字体字号 | 标题用黑体、正文用小四号 | 标题2号小标宋体，正文3号仿宋_GB2312 |
| 层次序号 | 1、（一）、一.、(1)等不规范 | 严格按照"一、（一）1.（1）"顺序 |
| 行距 | 自行设置行距 | 固定行距28磅 |
| 页码 | 未标注或标注位置错误 | 4号半角宋体，单右双左 |

### 二、逻辑类错误

| 错误类型 | 错误表现 | 正确做法 |
|---------|---------|---------|
| 层次不清 | 各部分内容交叉重复 | 按照逻辑关系重新划分，确保相对独立 |
| 逻辑混乱 | 因果倒置、前后矛盾 | 梳理逻辑关系，调整先后顺序 |
| 题文不符 | 标题不能概括内容 | 重新修改标题或调整内容 |
| 论证不足 | 观点缺乏论据支撑 | 补充数据、事例、政策依据 |

### 三、语言类错误

#### 1. 口语化严重

**错误**：使用方言、土语、网络语言

**示例**：
```
这事儿就这么定了，咱们得撸起袖子加油干！
```

**修改**：
```
会议决定事项，有关单位要抓好落实。
```

#### 2. 用词不准确

**常见错误用词**：

| 错误 | 问题 | 纠正 |
|------|------|------|
| 明火执仗 | 用词不当 | 公开、公然 |
| 首当其冲 | 误用为"首先" | 比喻最先受到攻击或遭遇伤害 |
| 差强人意 | 误用为"不满意" | 大体上还能使人满意 |
| 鼎力相助 | 感谢别人帮助 | 用于请托或感谢，不能用于自己帮助人 |
| 叹为观止 | 形容事物好到极点 | 不能用于负面事物 |

#### 3. 句式冗长

**错误示例**：
```
为了贯彻落实市委、市政府关于进一步优化营商环境的决策部署，切实解决企业在生产经营过程中遇到的困难和问题，经研究决定，建立健全企业联系服务机制。
```

**修改建议**：
```
为优化营商环境，解决企业实际问题，决定建立企业联系服务机制。
```

### 四、文种使用错误

| 错误类型 | 错误表现 | 正确做法 |
|---------|---------|---------|
| 请示与报告混淆 | 把请示写成报告，报告里请求批准 | 请示必须一文一事，明确请求事项 |
| 请示与函混淆 | 向无隶属关系机关发请示 | 不相隶属机关应使用函 |
| 通知与通报混淆 | 用通报传达执行性事项 | 传达执行性事项应使用通知 |
| 请示与申请混用 | 标题用申请、请求等 | 一律使用"请示"文种 |

## 写作进阶技巧

### 一、提升公文底蕴

1. **学习业务知识**
   - 熟悉本部门、本领域政策法规
   - 掌握专业术语和规范表述
   - 了解工作流程和基本要求

2. **积累素材库**
   - 建立政策文件库
   - 建立典型事例库
   - 建立数据资料库
   - 建立精彩语句库

3. **研读优秀范文**
   - 学习立意和思路
   - 学习框架和结构
   - 学习语言和表达

### 二、掌握写作套路

#### 1. 通知类
```
开头 = 依据 + 目的
主体 = 事项 + 要求
结尾 = 特此通知
```

#### 2. 请示类
```
开头 = 缘由（政策依据 + 实际情况）
主体 = 请示事项（具体明确）
结尾 = 妥否，请批示
```

#### 3. 报告类
```
开头 = 依据 + 过渡句
主体 = 做法 + 成效 + 问题
结尾 = 今后打算或特此报告
```

#### 4. 总结类
```
开头 = 背景 + 过渡句（成绩概述）
主体 = 具体做法 / 经验体会
结尾 = 问题 + 打算 / 认识体会
```

### 三、培养文字敏感

1. **朗读法**
   - 写完后大声朗读
   - 发现不通顺、不自然的地方
   - 体会语言的节奏和韵律

2. **比较法**
   - 同一内容，尝试多种表达方式
   - 选择最准确、最简洁、最得体的

3. **换位思考**
   - 站在读者角度审视
   - 是否清晰易懂
   - 是否符合身份和场合

### 四、建立检查清单

**完成公文后，对照检查**：

**内容检查**：
- [ ] 是否紧扣主题
- [ ] 观点是否正确
- [ ] 材料是否典型
- [ ] 数据是否准确
- [ ] 重点是否突出

**结构检查**：
- [ ] 层次是否清晰
- [ ] 逻辑是否严密
- [ ] 详略是否得当
- [ ] 前后是否连贯
- [ ] 首尾是否呼应

**语言检查**：
- [ ] 是否准确规范
- [ ] 是否简洁明了
- [ ] 是否平实庄重
- [ ] 有无错词病句
- [ ] 标点符号是否正确

**格式检查**：
- [ ] 文种使用是否正确
- [ ] 格式要素是否齐全
- [ ] 字体字号是否规范
- [ ] 排版打印是否美观

---

**总结**：公文写作是一项实践性很强的技能，需要在掌握基本要求的基础上，通过不断学习、勤于实践、善于总结来逐步提高。要养成严谨细致的工作作风，树立精益求精的质量意识，努力做到"文经我手无差错，事经我办请放心"。
